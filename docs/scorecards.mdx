Scorecards aggregate the results from your agent's level performance. The `Arcade.get_scorecard()` method returns an `EnvironmentScorecard` whose `score` property is the **HLG-RHAE** ([scoring methodology](/methodology)).

## Ways to get a scorecard

- **Manually** — see [Full Play Test](/full-play-test).
- **Python Toolkit** — see [Get Scorecard](/toolkit/get-scorecard).
- **Swarm** — running a [swarm](/swarms) automatically opens / closes one scorecard per agent.

## Scorecard fields

| Field           | Description                                                                                     |
| --------------- | ----------------------------------------------------------------------------------------------- |
| `tags`          | Strings used to categorize and filter scorecards (e.g. `["experiment1", "v2.0", "test"]`).      |
| `source_url`    | Optional URL field (e.g. your agent repo).                                                      |
| `opaque`        | Optional arbitrary JSON-serialisable data for downstream tools.                                 |
| `levels`        | `{level_id: [LevelResult, ...]}` recording every terminal event.                                |
| `score`         | HLG-RHAE computed lazily from `levels` against the optimal-length baseline.                     |

```python theme={null}
import hlg

arc = hlg.Arcade()
sc_id = arc.create_scorecard(
    tags=["experiment", "my-awesome-agent-v5-final-final"],
    source_url="https://github.com/my/repo",
    opaque={"custom_field": "any data"},
)

env = arc.make("level0", scorecard_id=sc_id)
# ... play ...

sc = arc.get_scorecard(sc_id)
print(sc.score, len(sc.games))
```

For more, see [Create Scorecard](/toolkit/create-scorecard).

## Sharing

In v0.1 (offline-only) scorecards are local. In v0.2 you'll be able to share replays from scorecards created via the API.

### Notes

- The default scorecard is created lazily on the first `make()` call without an explicit `scorecard_id`.
- Local scorecards are not auto-closed; call `arc.close_scorecard()` to finalize, or `to_jsonl(path)` to dump.
- HLG-RHAE on a partial scorecard reflects only the levels seen so far — handy for in-flight progress checks.
